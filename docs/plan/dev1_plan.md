# dev

基于2025年后端开发技术热点（云原生、AI融合开发、分布式架构等），结合市场需求和技术演进逻辑，制定以下分阶段学习计划，注重**实用性**与**就业竞争力**。计划分为**基础奠基 → 核心突破 → 高阶拓展**三个阶段，总周期建议 **6-8个月**。

## detail

---

### 📅 **阶段一：基础奠基（1-2个月）**  

**目标**：掌握编程核心能力与云原生基础，构建可落地的开发环境。  

| **技术领域**       | **学习内容与资源**                                                                 | **产出目标**                               |
|--------------------|------------------------------------------------------------------------------------|--------------------------------------------|
| **编程语言**       | ✅ **Go语言**（核心）：<br>  - 语法基础（协程、通道、接口）<br>  - 实战：开发CLI工具/HTTP服务（Gin框架）<br>  ✅ **Python**（辅助）：<br>  - 脚本编写 + Flask REST API开发 | Go实现高并发文件处理器<br>Python搭建AI模型调用API |
| **云原生入门**     | ✅ **Docker**：<br>  - 镜像构建/容器管理<br>  - Docker Compose多服务编排<br>  ✅ **Linux**：<br>  - 命令行操作、进程管理、网络配置        | 部署Python+Go双容器微服务                  |
| **数据基础**       | ✅ **SQL**（PostgreSQL）：<br>  - 复杂查询/事务/索引优化<br>  ✅ **Redis**：<br>  - 数据类型/持久化/缓存策略                         | 设计电商购物车（Redis）+订单系统（PostgreSQL）|

> **工具推荐**：  
>
> - Go实战：[《Go语言设计与实现》](https://golang.design/under-the-hood/)  
> - Docker实验室：[Play with Docker](https://labs.play-with-docker.com/)  
> - 数据库练习：[SQLBolt](https://sqlbolt.com/)  

---

### ⚡ **阶段二：核心突破（3-4个月）**  

**目标**：攻克企业级云原生架构与AI融合开发，匹配一线大厂需求。  

| **技术领域**         | **学习内容与资源**                                                                                | **产出目标**                               |
|----------------------|----------------------------------------------------------------------------------------------------|--------------------------------------------|
| **云原生架构**       | ✅ **Kubernetes**：<br>  - 集群部署/Service/Ingress<br>  - Helm包管理<br>  ✅ **Serverless**：<br>  - AWS Lambda/Azure Functions事件驱动开发 | 部署高可用Web服务（K8s + 自动扩缩容）       |
| **分布式系统**       | ✅ **微服务设计**：<br>  - gRPC通信/服务发现（Consul）<br>  - 熔断限流（Hystrix）<br>  ✅ **消息队列**：<br>  - Kafka/RabbitMQ订单削峰实战      | 实现分布式电商系统（商品/订单/支付微服务） |
| **AI赋能开发**       | ✅ **AI辅助工具链**：<br>  - GitHub Copilot代码生成<br>  - ChatGPT优化SQL/文档<br>  ✅ **AI运维**：<br>  - Prometheus+Grafana监控告警        | 用AI重构冗余代码 + 预测系统负载            |
| **安全与API**        | ✅ **JWT/OAuth2.0** 身份认证<br>  ✅ **RESTful/gRPC API设计**<br>  ✅ **DDos防御**（Nginx限流）                         | 实现权限管理系统（RBAC + JWT签发）         |

> **项目驱动**：  
>
> - 云原生电商平台：Spring Cloud Alibaba + K8s集群 + Redis缓存  
> - AI运维监控平台：Prometheus收集Go微服务指标 + Grafana可视化

### 🚀 **阶段三：高阶拓展（1-2个月）**  

**目标**：深入前沿技术与工程效能优化，提升架构决策能力。  

| **技术领域**       | **学习内容与资源**                                                                 | **应用场景**                             |
|--------------------|------------------------------------------------------------------------------------|------------------------------------------|
| **性能调优**       | ✅ **Go性能分析**（pprof + trace）<br>  ✅ **JVM调优**（GC算法/内存溢出排查）<br>  ✅ **数据库分库分表**（ShardingSphere）   | 优化10万QPS订单系统延迟                  |
| **边缘计算**       | ✅ **K3s轻量级K8s**<br>  - 边缘节点部署TensorFlow模型预测                            | 智能工厂设备实时数据分析                 |
| **量子加密探索**   | ✅ **QKD（量子密钥分发）原理**<br>  - 使用LibOQS实践抗量子签名算法                     | 金融系统安全升级方案预研                 |

> **持续增值方向**：  
>
> - 贡献CNCF开源项目（如etcd、TiKV）  
> - 考取CKA（Kubernetes认证）、AWS/Azure云架构师认证

### ⚠️ **计划执行关键原则**  

1. **二八法则聚焦**：  
   - 70%精力投入 **Go + K8s + 分布式设计**（当前最高ROI方向）  
   - 20%分配给AI工具链应用  
2. **避免“知识囤积”**：  
   - **每学完一个技术点，立刻用项目验证**（参考Github热门开源项目设计）  
   - 拒绝“只学不用”的教程循环  
3. **技术雷达更新**：  
   - 每周速览[Hacker News](https://news.ycombinator.com/) + [CNCF博客](https://www.cncf.io/blog/)
