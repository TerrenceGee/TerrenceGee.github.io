### **Semantic Versioning（语义化版本控制，简称 SemVer）详解**

Semantic Versioning 是一种**版本号命名规范**，用于明确代码库（如 Rust 的 crate）的版本升级对用户的影响。其核心是通过版本号的三段式结构（`MAJOR.MINOR.PATCH`）传达变更的**兼容性级别**，帮助开发者安全地管理依赖。

---

## **1. 版本号格式**

SemVer 的版本号由三部分组成，格式为：  
**`MAJOR.MINOR.PATCH`**  

- **`MAJOR`**（主版本号）：**不兼容的 API 变更**（用户需修改代码适配）。  
- **`MINOR`**（次版本号）：**向后兼容的功能新增**（用户无需修改代码）。  
- **`PATCH`**（修订号）：**向后兼容的问题修复**（用户无需修改代码）。  

**示例**：  

- `1.0.0`：初始稳定版本。  
- `1.2.3`：第 1 主版本，第 2 次版本，第 3 次修订。  

---

## **2. 关键规则**

### **（1）版本升级条件**

| **版本段** | **何时递增**                              | **用户影响**                     |
|------------|------------------------------------------|----------------------------------|
| `MAJOR`    | 破坏性变更（如删除函数、修改接口行为）。 | 必须检查代码兼容性，可能需要修改。 |
| `MINOR`    | 新增功能（不影响现有功能）。             | 可安全升级，无需修改代码。        |
| `PATCH`    | 修复 bug（不改变功能行为）。             | 应尽快升级，无风险。              |

### **（2）预发布版本与元数据**

- **预发布版本**：在正式版本前标记测试版（如 `1.0.0-alpha.1`）。  
- **构建元数据**：附加额外信息（如 `1.0.0+20230901`），不影响版本优先级。  

---

## **3. 在 Rust 中的实践**

### **（1）Cargo.toml 中的版本约束**

Rust 的包管理器 `cargo` 使用 SemVer 解析依赖，常见语法：  

```toml
[dependencies]
serde = "1.0"          # 允许 1.0.0 ≤ 版本 < 2.0.0（自动接受 MINOR/PATCH 更新）
tokio = "=1.29.1"      # 强制固定版本
axum = ">=0.6, <0.7"   # 指定范围
```

### **（2）版本冲突解决**

- **Cargo.lock**：二进制项目锁定依赖版本确保可重现构建。  
- **`cargo update`**：更新依赖版本（遵循 SemVer 约束）。  

---

## **4. 为什么 SemVer 重要？**

- **避免“依赖地狱”**：明确版本变更的兼容性，减少意外破坏。  
- **自动化升级**：工具可安全自动更新 `MINOR` 和 `PATCH` 版本。  
- **信任与透明**：用户通过版本号即可判断升级风险。  

---

## **5. 常见问题**

### **Q：`0.x.y` 版本是否遵循 SemVer？**  

A：`0.x.y` 被视为**初始开发阶段**，`MINOR` 版本升级可能包含破坏性变更（如从 `0.1.0` 到 `0.2.0`）。稳定版本从 `1.0.0` 开始。

### **Q：如何发布破坏性变更？**  

A：递增 `MAJOR` 版本（如 `2.0.0`），并提供迁移指南。对于 Rust 生态，可通过 `deprecated` 属性逐步弃用旧 API。

---

## **总结**

| **动作**               | **版本号变化**       | **用户应对措施**               |
|------------------------|----------------------|--------------------------------|
| 修复 bug               | `1.0.0` → `1.0.1`    | 无需修改代码，建议升级。       |
| 新增功能               | `1.0.0` → `1.1.0`    | 无需修改代码，可选择性使用。   |
| 破坏性变更             | `1.0.0` → `2.0.0`    | 必须检查兼容性，可能需修改代码。|

SemVer 是 Rust 生态健康发展的基石，严格遵守它能显著提升协作效率。
